🛡️ SnowBe Online – Security Plan & Risk Management Framework

📌 Project Overview

This project delivers a comprehensive Information Security Plan for SnowBe Online, a $100M lifestyle brand that recently went public. As the company transitioned from a laid-back startup to a publicly traded entity, it became essential to address critical gaps in its cybersecurity posture.

This plan follows the NIST 800-53r5 framework and leverages principles from the Risk Management Framework (RMF) and Cybersecurity Capability Maturity Model (CMMC) to protect data confidentiality, integrity, and availability across all systems.

🧭 Scope of Implementation

This security plan applies to:

  • All employees, contractors, and third-party vendors
  • Digital platforms hosted on AWS (e-commerce, databases)
  • Physical storefronts in the U.S. and Europe
  • Desktops, laptops, and mobile devices across the organization
  • On-premise and cloud-based servers

🛠️ Core Security Policies Implemented

Policy Area Controls & Focus
🔐 Access Control AC-1, AC-2, AC-3, AC-6, AC-15, AC-17, AC-19 – Least privilege, account management, remote/mobile security
🧪 System Integrity SI-2, SI-7 – Patch management, software/firmware protection
⚙️ Configuration Management CM-3 – Change control governance
🔑 Cryptography SC-12, SC-13 – Encryption at rest/in transit, key management
🚨 Incident Response IR-1 – Detect/respond to security incidents
💳 PCI Compliance AC-1 (PCI) – Cardholder data protection
🧠 Security Awareness AT-2 – Employee cyber hygiene and social engineering training
🔁 SDLC Security SDLC-1 – Secure development lifecycle (NIST SP 800-160, SSDF)

📋 Standards & Procedures

  • New Account Procedure
  • Password Management and Standards
  • Session Termination and Unsuccessful Logon Controls
  • Software Patch Management
  • CMMC Security Maturity Model Adoption

🔍 Risk Management Insights

Recommendations based on SnowBe case study:

  • Update firmware, OS patches, antivirus, backup systems
  • Restrict access rights on shared resources
  • Physically secure servers (locked room)
  • Archive login data to cloud storage
  • Approve and secure mobile devices

✅ Outcomes

  • A structured policy framework across governance, operations, and tech layers
  • Control mapping to NIST 800-53r5 for compliance
  • Implementation roadmap based on CMMC and RMF
